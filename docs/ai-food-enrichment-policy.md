# AI Food Enrichment Policy (MVP)

## Purpose
Define how unknown foods are handled safely when AI provides nutrient estimates, and how those results can be added to the product database.

## Definitions
- Known food: A food profile found in trusted nutrition dataset(s) with required nutrient fields.
- Unknown food: No confident match exists in local dataset.
- AI profile: Nutrient profile generated by AI assistant from user input.
- Confidence score: Model-level confidence (0-1) for generated profile quality and portion interpretation.

## Required Nutrient Schema (MVP)
AI output must include per 100g (or normalized equivalent):
- calories_kcal
- protein_g
- carbs_g
- fats_g
- total_sugars_g
- natural_sugars_g (if unavailable, return null with explanation)
- fiber_g
- sodium_mg
- source_notes

## Confidence Thresholds
- High confidence: >= 0.85
  - Auto-populate preview values.
  - User still must confirm before save.
- Medium confidence: >= 0.65 and < 0.85
  - Show warning badge and require user review/edit before save.
- Low confidence: < 0.65
  - Do not allow direct save.
  - Prompt user to refine item name, brand, or portion details.

## Save Policy
Unknown food can be persisted only when all are true:
- Confidence >= 0.65.
- Required nutrient schema is present (null allowed only for natural_sugars_g with explanation).
- User confirms save action explicitly.
- System writes provenance metadata.

## Provenance and Audit Fields
Every AI-created or AI-updated food profile must store:
- created_by: user_id
- generation_source: `ai_assistant`
- confidence_score
- model_id
- prompt_hash (non-PII hash)
- created_at and updated_at
- review_status (`user_confirmed`, `edited_after_confirm`, `flagged`)

## User Confirmation UX Requirements
- Show side-by-side summary:
  - Parsed food name + portion
  - Nutrient estimates
  - Confidence badge and confidence explanation
- Mandatory actions:
  - `Confirm and Save`
  - `Edit Values`
  - `Cancel`
- If edited, save as user-edited AI profile and track review status.

## Safety and Quality Rules
- AI must not return medical advice.
- AI must include uncertainty text when inferred from generic food categories.
- Reject impossible macro combinations (e.g., negative values, calories inconsistent with macros beyond tolerance).
- Validation tolerance:
  - kcal_from_macros = (protein_g * 4) + (carbs_g * 4) + (fats_g * 9)
  - Allow +/-15% difference before warning.

## Moderation / Abuse Controls
- Rate limit unknown-food AI calls per user/day (free-tier quota).
- Block suspicious prompt patterns intended to poison food DB.
- Add admin review queue for frequently used AI-added foods.

## Failure Handling
- If AI response invalid or below threshold:
  - Do not write food profile.
  - Keep user meal log as unresolved draft.
  - Ask user for more details or suggest closest known foods.

## Observability Events
- `ai_food_lookup_requested`
- `ai_food_lookup_completed`
- `ai_food_lookup_rejected_low_confidence`
- `ai_food_profile_confirmed`
- `ai_food_profile_edited_then_saved`
- `ai_food_profile_save_cancelled`
